◆ RelaxedSslValidation

bool CymaticLabs.Unity3D.Amqp.AmqpClient.RelaxedSslValidation = false

Whether or not to use relaxed SSL certificate validation for the broker.

This can be useful since Unity's version of Mono (as of 5.x) has a very strict/buggy implementation around SSL/TLS certificate validation. This can make the connection inherently less secure as it will trust unverified certificates, but encrypted communications will work. This setting is applied on start and updating it during runtime will have no effect.

◆ SubscribeRetryLimit

byte? CymaticLabs.Unity3D.Amqp.AmqpClient.SubscribeRetryLimit
getset

Gets or sets the maximum number of failed subscriptions the client will tolerate before preventing connection to the host.

Presently in RabbitMQ if there is an error during subscription the host will close the connection so the client must reconnect. In cases where the client attempts to resubscribe the same failing subscription this can lead to an endless loop of connect->subscribe->error->reconnect->infinity. Putting a limit prevents the loop from going on infinitely.
